# Yaskawa Robot Protocol (安川机器人协议) — 协议详情

> **格式参考**: 多协议通信矩阵、Modbus 实现缺陷、YMConnect 安全、控制器型号等细节见 [协议格式手册.md](协议格式手册.md)

## 协议演进

安川电机（Yaskawa Electric）是全球最大的工业机器人制造商之一，其 Motoman 系列机器人控制器（DX200、FS100、YRC1000 等）使用多协议栈进行通信，没有单一的"安川协议"——而是组合使用 Modbus TCP、EtherNet/IP、HSE 和 YMConnect。

## 协议栈

安川机器人控制器同时支持多种协议，用于不同场景：
- **Modbus TCP** (502): 标准工业数据交换，与 PLC/HMI 互通
- **EtherNet/IP** (44818): Rockwell 生态集成
- **HSE** (High Speed Ethernet): 实时伺服控制，UDP 自定义端口
- **YMConnect / MotoCom SDK**: 编程与配置，UDP 10040-10041

## 工控应用 — 工业机器人
安川 Motoman 系列广泛应用于：
- 汽车制造焊接与喷涂
- 半导体晶圆搬运
- 金属加工与弧焊

## 安全风险

### Modbus 实现缺陷
安川的 Modbus TCP 实现存在非标准行为（功能码混用、地址溢出），可能导致：
- 第三方 Modbus 工具无法正确读写
- 安全审计设备（IDS/IPS）难以解析和检测攻击流量

### YMConnect 默认凭据
- 默认 MANAGEMENT 密码: `999999`（全 9）
- 通过 MotoCom SDK 可远程上传/下载程序、启停机器人
- 建议将 UDP 10040-10041 限制到可信子网

### 物理安全影响
工业机器人的网络攻击可产生严重物理后果：非授权启停、运动轨迹篡改、碰撞风险。

## 相关标准
- Modbus TCP (IEC 61158) — 数据交换层
- EtherNet/IP (ODVA) — 数据交换层
- YMConnect / MotoCom SDK — 安川私有编程协议
