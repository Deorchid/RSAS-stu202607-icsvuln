# TRDP — 漏洞分析

## 漏洞概况

TRDP 协议在设计之初以**实时性和安全性**为首要目标，但缺乏基本的安全控制机制。

| 指标 | 数据 |
|------|------|
| 已知CVE | 较少（协议层面），主要通过研究论文披露 |
| 主要风险 | 无加密、无认证、无完整性校验 |
| 影响范围 | 全球轨道交通系统（高铁、地铁） |

## 协议层安全缺陷

| 缺陷 | 描述 | 风险等级 |
|------|------|---------|
| **无加密** | TRDP 数据明文传输，可被窃听和拦截 | 🔴 高危 |
| **无认证** | 任何网络设备都可伪造 TRDP 命令 | 🔴 高危 |
| **无完整性校验** | 数据帧无法检测篡改、重放、重排序 | 🔴 高危 |
| **UDP 泛洪风险** | Process Data 基于 UDP，可被泛洪攻击 | 🟡 中危 |
| **协议开源** | TCNOpen 项目开源，攻击者可逆向研究 | 🟡 中危 |
| **无访问控制** | 无发布/订阅权限限制 | 🟡 中危 |

## STRIDE 威胁模型分析

| 威胁类型 | 攻击向量 |
|---------|---------|
| **Spoofing（欺骗）** | 伪造 TRDP 命令、冒充终端设备 |
| **Tampering（篡改）** | 固件修改、恶意代码注入 |
| **Repudiation（抵赖）** | 日志删除和伪造 |
| **Information Disclosure（信息泄露）** | 数据包捕获暴露网络拓扑和控制数据 |
| **DoS（拒绝服务）** | TRDP 协议泛洪、设备资源耗尽 |
| **Elevation of Privilege（权限提升）** | 弱密码策略、维护端口利用 |

## 端口信息
- **Process Data**: UDP 17224
- **Message Data**: TCP 17225-17226

## 安全研究现状

| 研究项目 | 时间 | 描述 |
|---------|------|------|
| PHA 安全分析 | 2017 | 识别 TRDP 软件的36个初步危害（韩国研究） |
| TREADS 异常检测 | 2021 | 基于多流聚类的 TRDP 流量异常检测 |
| Seq2Seq 模糊测试 | 2023 | 基于深度学习的 TRDP 协议漏洞发现 |
| 加密方案设计 | 2025 | 针对 TRDP 过程数据和消息数据的加密协议 |

## 安全建议

1. **网络分段** — 在工业防火墙上严格限制 TRDP 端口访问
2. **主机防火墙** — 列车单元上只白名单可信源 IP
3. **IDS/IPS 签名** — Suricata 等检测 TRDP UDP 泛洪和畸形包
4. **协议感知监控** — 理解 TRDP 报文结构的异常检测
5. **应用层加密** — 在应用层添加认证和加密（IEC 61375 尚未标准化）
