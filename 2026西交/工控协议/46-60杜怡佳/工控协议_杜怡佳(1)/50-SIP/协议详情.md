# SIP (Session Initiation Protocol) — 协议详情

> **格式参考**: 请求/响应报文格式、方法定义、响应码分类等细节见 [协议格式手册.md](协议格式手册.md)

## 协议演进

SIP 由 IETF MMUSIC 工作组制定，旨在替代 H.323 成为 VoIP 的信令协议。

| 时间 | 里程碑 | 说明 |
|------|--------|------|
| 1999 | RFC 2543 | SIP 1.0 初始版本 |
| 2002 | RFC 3261 | SIP 2.0，当前主要版本 |
| 后续 | 数十个扩展 RFC | INFO、MESSAGE、REFER、PRACK 等方法扩展 |

## 协议栈

SIP 只负责**信令**（会话建立/修改/终止），媒体传输由 RTP/RTCP 承载。

## 协议特征
- **标准**: RFC 3261（核心）+ 大量扩展 RFC
- **端口**: 5060 (UDP/TCP), 5061 (TLS)
- **语法**: 类 HTTP 文本协议
- **端到端**: 中间 Proxy 可转发但不修改核心字段

## 工控与 IoT 应用
- 工业 VoIP 调度电话（如电厂/化工厂内部通信）
- 视频监控系统（SIP 作为 RTSP 的补充信令协议）
- 门禁对讲系统

## 安全问题
1. **明文传输**: 默认无加密，SIP Digest 认证可被离线破解
2. **DoS**: INVITE 洪泛、畸形包导致 SIP 服务器资源耗尽
3. **扫描/枚举**: OPTIONS 方法可用于发现活跃分机号
4. **劫持**: 中间人攻击篡改 SDP 协商，重定向 RTP 媒体流

## 相关标准
- RFC 3261 (SIP 2.0 核心)
- RFC 4566 (SDP)
- RFC 3550 (RTP/RTCP)
