# HID VertX Edge — 漏洞分析

## 漏洞概况

HID VertX/Edge 门禁控制器曾存在一个 **CVSS 10.0（临界）** 的命令注入漏洞，允许攻击者远程开锁。

| 指标 | 数据 |
|------|------|
| 最高 CVSS | **10.0（临界）** |
| 漏洞类型 | 命令注入（未授权） |
| 攻击向量 | UDP 4070 → 远程 root 命令执行 |
| 影响范围 | 机场、医院、政府设施、企业办公楼 |

## 重点漏洞

### CVE-未分配（ZDI 报告）— Remote Door Unlock（CVSS 10.0）

**发现者**: Ricky Lawshae（Trend Micro DVLabs，2016.02）
**漏洞组件**: `discoveryd` 服务（UDP 4070）

**利用方式**:
1. 向控制器 UDP 4070 发送特制数据包
2. 在 blink 命令参数中注入反引号包裹的 Linux 命令
3. 命令以 **root** 权限在控制器上执行
4. **无需认证**

**攻击后果**:
- 🔓 远程开锁/关锁
- 🚨 关闭警报
- 📋 下载持卡人数据库（全部门禁卡数据）
- 🔄 永久开锁（远程管理无法恢复）
- 💻 任意 root 命令执行

**公开工具**: [VertXploit](https://github.com/ColeShelly/VertXploit) — 自动利用框架

| 功能 | 说明 |
|------|------|
| discover | 发现局域网内的 HID 控制器 |
| fingerprint | 获取设备详细信息（MAC、固件、类型） |
| unlock/lock | 远程开/关锁 |
| raw | 以 root 执行任意 Linux 命令 |
| download | 导出持卡人数据库 |

## 补丁状态

- ✅ **已修复** — HID 已发布固件更新
- ❗ 但许多机构 **从未更新** 嵌入式控制器固件

## 安全建议

1. **更新固件** — 通过 HID 合作伙伴门户获取最新固件
2. **网络隔离** — 门禁控制网络与 IT 网络分离
3. **UDp 4070 过滤** — 在防火墙上限制 UDP 4070 访问
4. **IDS 规则** — Emerging Threats 已有 UDP 4070 扫描检测规则
5. **定期审计** — 检查控制器固件版本和暴露面
