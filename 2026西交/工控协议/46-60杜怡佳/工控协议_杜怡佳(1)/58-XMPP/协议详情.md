# XMPP (Extensible Messaging and Presence Protocol) — 协议详情

> **格式参考**: 协议栈分层、Stanza XML 示例、IoT XEP 扩展列表等细节见 [协议格式手册.md](协议格式手册.md)

## 协议演进

XMPP 前身为 Jabber（1999 年），2004 年由 IETF 标准化为 RFC 3920/3921，2011 年更新为 RFC 6120/6121。XMPP Standards Foundation (XSF) 维护大量 IoT 扩展 (XEP)。

## 协议栈

基于 XML 流的异步消息协议，客户端 TCP 5222，服务器间 TCP 5269。核心分层：TCP → TLS (STARTTLS) → SASL (认证) → XMPP Stanza (XML 消息)。

## 协议特征
- **标准**: IETF RFC 6120/6121 + XSF XEP 系列扩展
- **模型**: 去中心化联邦架构（类似 Email）
- **编码**: XML Stream（SASL 认证后的 XML 元数据流）
- **安全**: STARTTLS 传输加密 + SASL 多种认证机制 + Channel Binding 防中间人

## 工控应用 — ABB 分布式控制系统

XMPP 在工业自动化领域最知名的应用是 **ABB Ability™** 和 **800xA** 控制系统：
- 分布式控制中心通过 XMPP 进行实时消息交换
- `<message/>` 用于告警推送（如温度超标、设备离线）
- `<presence/>` 用于设备在线/离线状态通告
- `<iq/>` 用于传感器数据查询（XEP-0323 IoT Sensor Data）

XMPP 的发布/订阅模型天然适合 SCADA 系统的遥测数据分发。

## 安全保障
- **TLS**: STARTTLS 加密传输
- **SASL**: SCRAM-SHA-1/PLUS 多种认证
- **Channel Binding**: 将 TLS 会话绑定到 SASL 认证，防止 MITM 中继攻击
- **Stanza 过滤**: 服务端过滤 XML 注入和恶意 Stanza

## 相关标准
- RFC 6120 (XMPP Core) / RFC 6121 (XMPP IM)
- XEP-0323/0324/0325/0326 (IoT 传感器与控制扩展)
- ABB Ability™ / 800xA 系统文档
