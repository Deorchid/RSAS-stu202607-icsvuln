# UMAS (Unity Modbus Application Protocol) — 协议详情

> **格式参考**: Modbus 功能码封装、认证版本演进、已知 CVE 列表等细节见 [协议格式手册.md](协议格式手册.md)

## 协议演进

UMAS 是 Schneider Electric 为其 Modicon Unity 系列 PLC 设计的私有配置协议，运行在 Modbus TCP 502 端口之上，使用 Modbus 功能码 0x5A (90) 封装。

UMAS 存在是因为标准 Modbus 只能读写寄存器——它无法完成 PLC 配置、编程、固件更新等高级操作。UMAS 填补了这个缺口，但同时引入了巨大的安全风险。

## 协议栈

```
┌─────────────────┐
│ UMAS (功能码0x5A)│ ← Schneider 私有扩展
├─────────────────┤
│ Modbus TCP      │ ← 标准 Modbus, 端口 502
├─────────────────┤
│ TCP / IP        │
└─────────────────┘
```

## 安全研究里程碑

UMAS 是工控安全领域被研究最深入的私有协议之一：

| 年份 | 研究者 | 发现 |
|------|--------|------|
| 2017 | ClearEnergy (Reid Wightman) | 首次公开披露 UMAS 设计缺陷 (CVE-2017-6032) |
| 2020 | 卡巴斯基 ICS-CERT | 完整逆向分析 UMAS 协议结构 |
| 2021-22 | Forescout / OT:ICEFALL | 多个高危 CVE (RCE, 认证绕过) |

## 核心安全缺陷
1. **认证可绕过**: 低固件版本使用固定 Key 或 1 字节 Session Key (0-255)，可暴力破解
2. **Nonce 重用** (CVE-2021-22779, CVSS 9.8): 认证协议设计缺陷，可完全绕过认证
3. **RCE** (CVE-2022-45788): 0x50 命令远程代码执行
4. **隐蔽性**: 运行在标准 Modbus 502 端口，流量混在正常 Modbus 通信中，难以检测

## 工控应用
- Schneider Modicon M340 / M580 / Quantum 系列 PLC 的工程配置和维护
- EcoStruxure Control Expert (原 Unity Pro) 编程软件通过 UMAS 与 PLC 交互

## 相关标准
- Modbus TCP (IEC 61158) — 承载层
- UMAS — Schneider Electric 私有扩展（非公开标准）
