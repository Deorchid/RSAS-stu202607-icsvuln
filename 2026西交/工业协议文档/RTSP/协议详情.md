# RTSP (Real Time Streaming Protocol) — 协议详情

> **格式参考**: 报文结构、方法定义、状态机等细节见 [协议格式手册.md](协议格式手册.md)

## 协议演进

| 时间 | 版本 | 说明 |
|------|------|------|
| 1998 | RTSP 1.0 (RFC 2326) | 初始版本 |
| 2016 | RTSP 2.0 (RFC 7826) | 更新版本 |

## 协议栈

```
┌─────────────────┐
│   RTSP (控制)    │ ← TCP 554
├─────────────────┤
│   RTP (媒体)     │ ← UDP 动态端口
├─────────────────┤
│   TCP / UDP     │
├─────────────────┤
│   IP            │
└─────────────────┘
```

## 协议规范

RTSP 由 IETF 定义为 RFC 2326（1998年），后续更新为 RFC 7826（2016年）。

### 基本特征
- **用途**: 控制流媒体服务器（播放、暂停、录制）
- **语法**: 类 HTTP 文本协议（方法、头部、状态码）
- **状态**: RTSP 维护会话状态（不同于 HTTP 的无状态）
- **传输**: RTP 承载媒体数据，RTSP 控制流
- **默认端口**: TCP/UDP 554

### 工控应用
在工业视频监控系统中，IP 摄像头通过 RTSP 传输实时监控画面。
RTSP 结合 ONVIF 标准用于摄像头发现和管理。

### 安全问题
1. RTSP 基本认证使用 Base64 编码，易被解码
2. 缓冲区溢出漏洞频发（尤其在帧处理中）
3. 缺乏加密传输（可使用 RTSP over TLS）

## 相关标准
- RFC 2326 / RFC 7826
- ONVIF (Open Network Video Interface Forum)
