# S7COMM (Siemens S7 Communication) — 协议详情

> **格式参考**: TPKT/COTP头部结构、S7COMM PDU字段、地址编码、存储区代码等细节见 [协议格式手册.md](协议格式手册.md)

## 协议演进

S7COMM 随西门子 S7 系列 PLC 发展而来，从 S7-200 到 S7-1500 均有支持，是工业自动化领域最广泛使用的私有协议之一。

## 协议栈

S7COMM 运行在 TCP 102 端口，协议栈自下而上为：**IP → TCP → TPKT (RFC 1006) → COTP (ISO 8073) → S7COMM**。

### 连接建立过程（COS）
1. **CR** (Connection Request) — 客户端发起连接请求，携带 TSAP 参数标识机架/槽位
2. **CC** (Connection Confirm) — 服务器确认连接
3. **Setup Communication** — S7COMM 层握手（功能码 0xF0）
4. **数据传输** — Read/Write 等 S7 PDU 交换
5. **断开** — 连接释放

### 主要服务
- **S7 Read** — 读取 DB 数据块、输入(I)、输出(Q)、标志位(M) 等
- **S7 Write** — 写入数据到 PLC 存储区
- **S7 Request** — 获取 PLC 状态信息
- **S7 Block Info** — 获取程序块信息（OB/FC/FB/DB 列表）
- **S7 Control** — PLC 启停控制 (STOP/RUN)

### 工控安全风险
S7COMM 缺乏认证和加密机制，攻击者一旦接入网络即可：
- 读写 PLC 内部数据（篡改工艺参数）
- 发送 STOP 命令导致生产停机
- 上传恶意程序块实现持久化
