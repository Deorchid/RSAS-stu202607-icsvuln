# Sinec H1 协议格式手册

## 1. 协议概述

Sinec H1（Siemens Network Architecture）是西门子推出的第一个基于以太网的工业协议，基于 ISO/IEC 8073 标准定义传输层。

| 属性 | 值 |
|------|-----|
| 厂商 | Siemens |
| 物理层 | IEEE 802.3（Ethernet），10 Mbit/s，基带传输 |
| 传输层 | ISO 8073 Class 4（OSI 第4层） |
| 网络层 | 未启用 |
| 应用层 | SINEC AP 1.0 / TF (Technology Function) |
| 拓扑 | 总线型，50Ω同轴电缆 |
| 网段长度 | 最大 500 米 |
| 每段节点数 | 最大 100 |
| 介质访问 | CSMA/CD（碰撞检测） |

### 通信模式

| 模式 | OSI层 | 说明 |
|------|-------|------|
| PLC ↔ PLC | 第4层传输层连接 | 对等数据传输 |
| 上位机 ↔ PLC | 第7层 TF 通信 | 数据采集和控制，支持读写操作 |

### 硬件支持

Simatic S5/S7 系列 PLC 配合以下通信模块：
- CP1430, CP143, CP535（S5 系列）
- CP443-1（S7 系列）

编址方式：TSAP 节点、访问块、word 形式、数据长度

## 2. 协议栈结构

| OSI 层 | 协议 |
|--------|------|
| 应用层 (L7) | SINEC AP 1.0 / TF (Technology Function) |
| 表示层 (L6) | — |
| 会话层 (L5) | — |
| 传输层 (L4) | ISO 8073 Class 4 |
| 网络层 (L3) | 未启用 |
| 链路层 (L2) | IEEE 802.2 LLC / 802.3 MAC |
| 物理层 (L1) | IEEE 802.3 10Base5 / 10Base2 |

## 3. 报文帧结构

Sinec H1 报文通过逆向 SCADA 驱动（s7drv）和 Wireshark 插件源码分析得出。

### 3.1 总线连接报文格式

```
+------------------+
| 过程启动         |  SINEC H1 过程 (20 字节)
+------------------+
| 协议头           |  AP 1.0 头部 (8 字节) + AS 512 头 (2 字节)
+------------------+
| 标识             |  2 字节
+------------------+
| 错误号           |  2 字节
+------------------+
| 用户数据         |  最大 244 字节（FMS 连接时最大 224 字节）
+------------------+
| 过程终止         |  SINEC H1 过程 (2 字节)
+------------------+
```

### 3.2 请求帧结构（基于逆向分析）

| 字段 | 长度 | 说明 |
|------|------|------|
| 固定标识 | 可变 | 协议标识符/长度标记 |
| 操作类型 | 1B | Block Type（见 4.1） |
| 功能码 | 1B | Opcode（见 4.2） |
| 数据类型 | 1B | Organization（见 4.3） |
| DB块号 | 2B | 目标数据块编号 |
| 地址偏移 | 2B | 起始地址 |
| 数据长度 | 2B | 读取/写入长度 |

### 3.3 响应帧结构

| 字段 | 长度 | 说明 |
|------|------|------|
| 固定标识 | 可变 | 协议标识符/长度标记 |
| 操作类型 | 1B | Block Type（响应标记） |
| 子块类型 | 1B | 操作类型 + 响应标记 |
| 状态 | 1B | 执行状态 |
| 数据 | 可变 | 返回数据 |

## 4. 关键字段定义

### 4.1 Block Type（操作类型）

Wireshark `block_type_vals[]` 定义：

| 值 | 类型 |
|----|------|
| 0x01 | 请求 (Request) |
| 0x03 | 响应 (Response) |
| 0x04 | 错误 (Error) |
| 0x05 | 确认 (Acknowledge) |

### 4.2 Opcode（功能码）

| 值 | 功能 |
|----|------|
| 0x01 | 读 (Read) |
| 0x02 | 写 (Write) |
| 0x03 | 读响应 (Read Response) |
| 0x04 | 写响应 (Write Response) |

### 4.3 Organization（数据类型/区域）

对应访问 PLC 中的各种数据块：

| 值 | 区域 |
|----|------|
| 0x01 | 数字输入 (Digital Input) |
| 0x02 | 数字输出 (Digital Output) |
| 0x03 | 标志位 (Flag/Marker) |
| 0x04 | 数据块 (Data Block / DB) |
| 0x05 | 定时器 (Timer) |
| 0x06 | 计数器 (Counter) |
| 0x07 | 外设输入 (Peripheral Input) |
| 0x08 | 外设输出 (Peripheral Output) |
| 0x09 | 系统数据 (System Data) |

## 5. 通信流程

### 上位机轮询示例

1. 建立传输层连接（ISO 8073 Class 4）
2. 发送读取 DB 数据请求帧（指定 DB 块号、地址偏移、长度）
3. 接收 PLC 响应帧（含数据负载）
4. 发送写请求帧（指定 DB 块号、地址、数据）
5. 接收写确认响应
6. 断开连接

### Wireshark 解析示例

典型轮询 WRFlag 状态的请求帧：
- 操作类型: 读请求
- DB 块: DB38
- 地址: 0x0000
- 长度: 0x0001

典型读取 DB36 数据的请求帧：
- 操作类型: 读请求
- DB 块: DB36
- 地址: 0x0000
- 长度: 0x02EF（751 字节）
