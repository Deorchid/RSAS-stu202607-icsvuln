# UMAS — 漏洞分析

## 漏洞概况

UMAS 是 Schneider Electric 基于 Modbus TCP（功能码 0x5A）的私有协议，用于配置和编程 Modicon Unity 系列 PLC。

| 指标 | 数据 |
|------|------|
| 已知 CVE | **至少 6 个严重漏洞** |
| 最高 CVSS | **10.0（临界）** |
| 影响范围 | Modicon M340/M580/Quantum/Premium/Momentum 全系列 |
| 全球暴露 | 超过 1000 台 PLC 暴露在互联网 |

## 重点漏洞清单

### CVE-2021-22779（CVSS 9.8）— 认证绕过
- **描述**: 即使启用了 Application Password，nonce 重用仍允许重放攻击
- **影响**: M340 (pre-3.50), M580 (pre-4.02)
- **机理**: 所有密码学计算在客户端完成，相同的 nonce 产生相同的响应

### CVE-2022-45789（CVSS 8.1）— Session 劫持
- **描述**: 攻击者嗅探 nonce 后可伪造认证会话
- **影响**: M340, M580, M580 Safety, MC80, Momentum Unity, Quantum Unity, Premium Unity
- **机理**: Nonce 维护为全局变量，无客户端绑定，session 结束前不刷新

### CVE-2022-45788（CVSS 7.5）— 远程代码执行
- **描述**: 通过未公开的 UMAS 命令（service code 0x50）直接操作内存
- **影响**: 同上
- **机理**: service code 0x50 允许直接读/写/复制/调整内存块 —— 无需重启 PLC 即可"热补丁"

### CVE-2020-28212（CVSS 8.1）— Session Key 暴力破解
- **描述**: 未启用 Application Password 时，session key 仅 1 字节（0-255）
- **机理**: 固件 <2.7 时 key 恒为 0x01

### CVE-2017-6032 / CVE-2017-6034 — 协议设计缺陷
- **描述**: UMAS 协议基础设计缺陷，可实现勒索软件攻击
- **工具**: ClearEnergy/UMASploit（Proof of Concept）

## 安全建议

1. **升级固件** — M340 ≥3.50, M580 ≥4.02
2. **启用 Application Password**
3. **网络分段** — 限制端口 502/TCP 访问
4. **使用 ACL** — 在以太网模块上限制 IP 访问
5. **使用 IPSec** — M580 新版本支持（有限实现）
6. **监控 UMAS 流量** — 使用 OT 专有 IDS 签名
