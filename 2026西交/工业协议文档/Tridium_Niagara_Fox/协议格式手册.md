# Tridium Niagara Fox 协议格式手册

## 1. 协议概述

Fox 是 Tridium 公司 Niagara Framework 的专有通信协议，用于 Niagara Station、Workbench 和 Supervisor 之间的通信。

| 属性 | 内容 |
|------|------|
| 协议 | Fox（私有） |
| 端口（未加密） | TCP 1911 |
| 端口（TLS 加密） | TCP 4911（FoxS） |
| 传输层 | TCP / TLS |
| 认证 | SCRAM-SHA (n4digest) |
| 通信模式 | 复用式点对点 (Multiplexed Peer-to-Peer) |
| 数据模型 | 树型 ORD（Object Resource Descriptor） |

---

## 2. 协议特性

### 2.1 传输层

- 基于 TCP 的流协议
- 支持 TLS 加密（FoxS = Fox over TLS）
- 建议禁用 SSLv3，使用 TLS 1.2+

### 2.2 多路复用

Fox 协议内建 **分帧机制 (Framing)**，支持在同一 TCP 连接上多路复用多个请求/响应对，避免队头阻塞：
- 大响应被拆分为多个小块交错发送
- 每个请求/响应对有唯一标识

### 2.3 对象寻址

使用 **ORD (Object Resource Descriptor)** 标识 Niagara 对象树中的节点：
```
fox://username:password@ipaddress:port
foxs://username:password@ipaddress:port
```

示例: `fox://admin:pass@192.168.1.100:1911`

### 2.4 认证机制

| 版本 | 认证方式 | 说明 |
|------|---------|------|
| 旧版 | MD5 Digest | 较弱 |
| 新版 | SCRAM-SHA (n4digest) | 盐化挑战响应机制，与 RFC 5802 兼容 |

## 3. 通信流程

```
+----------+                  +-------------+
| 客户端   |                  | Niagara     |
| (Normal) |                  | Station     |
+----+-----+                  +------+------+
     |                               |
     | 1. TCP 连接 (1911/4911)       |
     |------------------------------>|
     |                               |
     | 2. TLS 握手 (FoxS 模式)       |
     |<=============================>|
     |                               |
     | 3. SCRAM-SHA 认证             |
     |<=============================>|
     |                               |
     | 4. 请求根 ORD (getRoot)       |
     |------------------------------>|
     | 5. 返回根节点 ORD             |
     |<------------------------------|
     |                               |
     | 6. 请求子节点列表             |
     |------------------------------>|
     | 7. 返回节点(点)树             |
     |<------------------------------|
     |                               |
     | 8. 订阅/轮询数据变化          |
     |<=============================>|
     |                               |
```

---

## 4. 功能支持

Normal 框架实现的 Fox 子集支持以下操作：

| 功能 | 说明 |
|------|------|
| 加载对象数据库 | 从 Niagara Station 加载完整对象树 |
| 点趋势 | 通过轮询或 CoV（Change of Value）订阅 |
| 点写入 | NumericWritable / BooleanWritable / EnumWritable |
| 节点探索 | 基于白名单的 Java 类节点类型过滤 |

### 节点探索白名单

Normal 只探索已白名单化的 Java 类节点类型，避免不兼容类型的意外加载。

## 5. 安全加固

| 措施 | 说明 |
|------|------|
| 强制 FoxS | 禁用未加密的 Fox，启用 TLS |
| TLS 版本 | 禁用 SSLv3，使用 TLS 1.2+ |
| 证书 | 替换默认自签名证书为 PKI 证书 |
| 认证模式 | 使用 Strict 模式（仅 SCRAM-SHA） |
| 网络隔离 | 不将 Fox/FoxS 端口暴露到互联网 |
| 最小权限 | 创建专用 Niagara 用户，限制 RBAC 权限 |
