# Tridium Niagara Fox Protocol — 协议详情

> **格式参考**: 端口信息、FoxS 加密模式、认证机制等细节见 [协议格式手册.md](协议格式手册.md)

## 协议演进

Fox Protocol 是 Tridium Niagara Framework 的私有通信协议。Niagara 是楼宇自动化领域最广泛使用的集成平台之一（2015 年被 Honeywell 收购）。

## 协议栈

基于 TCP 的复用式点对点协议，支持在一个 TCP 连接内多路复用多个逻辑会话。

## 协议特征
- **标准**: Tridium 私有协议
- **端口**: TCP 1911 (明文), TCP 4911 (FoxS, TLS 加密)
- **认证**: MD5 Digest (旧版) → SCRAM-SHA / n4digest (新版)
- **权限模型**: RBAC 三级 (Read / Write / Invoke)

## FoxS 安全增强
FoxS = Fox over TLS，为通信提供传输层加密和完整性保护。

## 工控应用 — 楼宇自动化
Niagara Framework 广泛应用于：
- 智能楼宇 HVAC（暖通空调）控制
- 能源管理系统
- 门禁与安防集成
- 数据中心环境监控 (DCIM)

## 安全风险
1. **明文 Fox (1911)**: 默认非加密模式，凭据和指令可被嗅探
2. **弱认证遗留**: 旧版 MD5 Digest 已不再安全
3. **暴露面大**: 楼宇自动化系统常与管理网络连通，成为 OT/IT 边界的薄弱环节
4. **供应链风险**: Niagara 作为集成平台，下层接入大量第三方协议（BACnet、Modbus 等），任一子协议漏洞可被利用
