# UPnP — 漏洞分析

## 漏洞概况

UPnP 在工控/OT 环境中是一个**被严重低估的威胁**。虽然 UPnP 本身不是工控协议，但它广泛存在于工业路由器、交换机和 IIoT 网关中。

| 指标 | 数据 |
|------|------|
| 已知 CVE | 数十个（持续增长） |
| 最新 CVE | CVE-2026-3622, CVE-2026-41682, CVE-2026-32077 |
| 主要风险 | 无认证端口映射、外部暴露、横向移动 |
| 影响范围 | 工业路由器、IIoT 网关、现场设备 |

## 关键漏洞

### CVE-2026-3622 — UPnP 越界读取
- **描述**: TL-WR841N 路由器的 UPnP 组件存在输入验证缺陷，导致拒绝服务
- **影响**: 工业/企业边缘路由器

### CVE-2026-41682 — pupnp SDK SSRF（CVSS 6.9）
- **描述**: `parse_uri()` 中 `atoi()` 转换导致端口截断，可进行 SSRF
- **影响**: 使用 pupnp 库的所有设备（大量嵌入式/工控设备）

### CVE-2026-32077 — Windows UPnP 提权（CVSS 7.8）
- **描述**: Windows UPnP Device Host 允许本地攻击者获得 SYSTEM 权限
- **影响**: OT 环境中的 Windows 工作站和服务

### CVE-2011-4504 — IGD 外部端口映射
- **描述**: UPnP IGD 实现允许远程攻击者在 WAN 接口建立端口映射
- **影响**: 打破 IT/OT 网络隔离

## UPnP 在工控环境的特殊风险

| 风险 | 说明 |
|------|------|
| **无认证** | UPnP 添加/修改端口映射不需要任何认证 |
| **外部暴露** | 许多工业路由器默认在 WAN 接口启用 UPnP |
| **横向移动跳板** | 攻破 UPnP 设备后可进入 Level 1/0 设备 |
| **不可代理** | OT 设备无法安装安全代理，UPnP 漏洞难被发现 |
| **拓扑泄露** | UPnP 响应会泄露内网拓扑信息 |

## 安全建议

| 措施 | 说明 |
|------|------|
| **禁用 UPnP** | 在工业路由器和网关上关闭 UPnP |
| **网络分段** | 使用防火墙和 VLAN 隔离 OT 网络 |
| **补丁管理** | 更新 UPnP 库到 ≥1.18.5 |
| **异常检测** | 监控异常的 UPnP AddPortMapping SOAP 请求 |
| **出站过滤** | 防止 UPnP 响应泄露网络拓扑 |
