# Yaskawa Robot — HSES 协议格式手册

## 1. 协议概述

HSES (High-Speed Ethernet Server) 是安川机器人控制器（DX100/DX200/FS100/YRC1000 等）的专有通信协议，用于上位机对机器人进行实时控制和文件管理。

| 属性 | 值 |
|------|-----|
| 传输层 | UDP |
| 机器人控制端口 | 10040 |
| 文件控制端口 | 10041 |
| 字节序 | Little-Endian |
| 最大报文长度 | 511 字节（头 32B + 负载 ≤479B） |
| 协议标识 | Magic = `YERC` |

---

## 2. 报文结构

```
+---------------------------+
| Header (32 bytes)         |
+---------------------------+
| Payload (0-479 bytes)     |
+---------------------------+
```

### 2.1 固定头 (32 字节)

| 偏移 | 大小 | 字段 | 说明 |
|------|------|------|------|
| 0–3 | 4 | Magic | `0x59 0x45 0x52 0x43`（ASCII "YERC"） |
| 4–5 | 2 | Header Size | 固定 `0x0020`（32） |
| 6–7 | 2 | Payload Size | 负载长度，可变 |
| 8 | 1 | Reserved | 固定 `0x03` |
| 9 | 1 | Division | `0x01` = 机器人控制, `0x02` = 文件管理 |
| 10 | 1 | ACK | `0x00` = 请求, `0x01` = 响应 |
| 11 | 1 | Request ID | 会话内递增编号 |
| 12–15 | 4 | Block Number | 块编号（见 2.2） |
| 16–23 | 8 | Reserved | ASCII `"99999999"` |
| 24–31 | 8 | Sub-header | 命令/响应子头 |

### 2.2 Block Number 规则

| 场景 | 值 |
|------|-----|
| 请求 | 固定 `0` |
| 单包响应 | `0x80000000` |
| 多包响应（非最后一包） | 从上一包递增 |
| 多包响应（最后一包） | 上一包值 + `0x80000000` |
| ACK 包 | 与对应数据包相同 |

---

## 3. 子头结构

### 3.1 请求子头 (Bytes 24–31)

| 偏移 | 大小 | 字段 | 说明 |
|------|------|------|------|
| 24–25 | 2 | Command ID | 命令编号 |
| 26–27 | 2 | Instance | 实例编号 |
| 28 | 1 | Attribute | 属性 |
| 29 | 1 | Service | 服务代码 |
| 30–31 | 2 | Padding | `0x0000` |

### 3.2 响应子头 (Bytes 24–31)

| 偏移 | 大小 | 字段 | 说明 |
|------|------|------|------|
| 24 | 1 | Service + 0x80 | 服务代码的最高位置1 |
| 25 | 1 | Status | 状态码 |
| 26 | 1 | Added Status Size | 附加状态长度 |
| 27 | 1 | Padding | `0x00` |
| 28–29 | 2 | Added Status | 附加错误码 |
| 30–31 | 2 | Padding | `0x0000` |

### 3.3 响应状态码

| 值 | 含义 |
|----|------|
| `0x00` | 正常 |
| `0x08` | 命令未定义 |
| `0x09` | 无效元素编号 |
| `0x1F` | 异常（检查 Added Status） |
| `0x28` | 实例不存在 |

---

## 4. 服务代码 (Service Codes)

通过 Service 字段选择操作类型（位于请求子头的 Byte 29）：

| 服务码 | 操作 |
|--------|------|
| `0x01` | Get Attribute |
| `0x02` | Set Attribute |
| `0x05` | Call (方法调用) |
| `0x08` | Get Attribute (多实例) |
| `0x0C` | Start |
| `0x0D` | Stop |
| `0x0E` | Reset |

Attribute 字段（Byte 28）进一步限定访问范围：
| 值 | 含义 |
|----|------|
| `0x30` | AI (Analog Input) |
| `0x31` | AO (Analog Output) |
| `0x41` | DI (Digital Input) |
| `0x42` | DO (Digital Output) |
| `0x64` | Robot Position |
| `0x65` | Robot Status |

## 5. 通信流程示例（读取机器人状态）

**请求报文 (Client → Controller):**
```
YERC            # Magic, 4 bytes
20 00           # Header Size = 32
?? ??           # Payload Size (小端)
03              # Reserved
01              # Division = Robot
00              # ACK = Request
01              # Request ID = 1
00 00 00 00     # Block Number = 0
39 39 39 39     # Reserved "99999999"
39 39 39 39     # Reserved "99999999"
65 00           # Command ID
01 00           # Instance = 1
65              # Attribute = Robot Status
01              # Service = Get Attribute
00 00           # Padding
```

**响应报文 (Controller → Client):**
```
YERC            # Magic
20 00           # Header Size
?? ??           # Payload Size (小端)
03              # Reserved
01              # Division = Robot
01              # ACK = Response
01              # Request ID = 1 (回显)
00 00 00 80     # Block Number = 0x80000000
39 39 39 39     # Reserved
39 39 39 39     # Reserved
81              # Service | 0x80
00              # Status = OK
00              # Added Status Size = 0
00              # Padding
00 00           # Added Status = 0
00 00           # Padding
[payload...]    # 机器人状态数据
```

## 6. 其他通信协议

| 协议 | 端口 | 用途 |
|------|------|------|
| Modbus TCP | TCP 502 | 数据交换（注意:安川实现有功能码不匹配/地址溢出等非标问题） |
| Ethernet/IP | TCP 44818 | 数据交换 |
| YMConnect (MotoCom SDK) | UDP 10040–10041 | 编程配置（默认密码全 9） |
| HSES (本手册) | UDP 10040–10041 | 实时控制 |
