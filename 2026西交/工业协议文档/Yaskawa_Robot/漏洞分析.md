# Yaskawa Robot — 漏洞分析

## 漏洞概况

安川（Yaskawa）Motoman 系列工业机器人的通信协议实现存在若干安全问题。

| 指标 | 数据 |
|------|------|
| 已知漏洞 | 协议实现缺陷（非CVE类） |
| 主要风险 | Modbus 实现不兼容、默认凭证、端口暴露 |
| 影响范围 | DX200/FS100/YRC1000 等控制器 |

## 协议实现缺陷

### 1. Modbus/TCP 实现违规
- **功能码不匹配**: Motoman 离散输入区域用 FC01（Read Coils）而非标准的 FC02（Read Discrete Inputs）
- **地址范围溢出**: 寄存器范围 70010–79997 和 80010–80647 超出 Modbus PDU 16位地址限制（最大 65535）
- **6位地址冲突**: 存储区标签前缀（5xxxx/6xxxx/7xxxx）与标准 Modbus 4xxxx/3xxxx/1xxxx 不兼容

### 2. YMConnect 协议安全问题
- **端口**: UDP 10040-10041
- **默认密码**: MANAGEMENT 安全等级默认密码为全 9（`999999`）
- **非实时**: 标准 Ethernet/UDP 不是确定性网络，不适合实时控制

### 3. HSE（High Speed Ethernet）协议
- **Request ID 漏洞**: RoboDK 驱动忽略 Request ID 字段，多应用并发时产生状态冲突

## 攻击面分析

| 攻击面 | 风险 | 说明 |
|--------|------|------|
| Modbus 端口 502 暴露 | 🟡 中危 | 地址映射异常导致 DoS |
| UDP 10040-10041 | 🟡 中危 | 默认密码可被暴力破解 |
| 管理员权限占比 90% | 🟡 中危 | 开发环境安全风险 |
| 老旧 jQuery | 🟢 低危 | 已知漏洞的 Web 组件 |

## 安全建议

| 问题 | 建议 |
|------|------|
| 不可达寄存器 | 使用 EtherNet/IP 或 DeviceNet 替代 Modbus |
| 功能码不匹配 | 配置客户端使用 FC01 而非 FC02 |
| 暴露端口 | 限制 UDP 10040/10041 和 TCP 502 到可信子网 |
| 默认密码 | 更改 MANAGEMENT 默认密码 |
| 固件更新 | 关注安川安全公告 |
